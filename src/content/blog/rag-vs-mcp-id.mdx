## Pendahuluan

Sistem AI modern sering menghadapi dua tantangan besar:
1. Cara memberikan jawaban yang akurat berdasarkan data nyata.
2. Cara berinteraksi dengan aman dengan alat dan API eksternal.

Dua konsep ada di pusat solusi ini: **Retrieval-Augmented Generation (RAG)** dan **Model Context Protocol (MCP)**.
Mereka terkait tetapi memecahkan masalah yang berbeda. Memahami peran mereka membantu menjelaskan bagaimana agen AI saat ini dibangun.

## Apa itu RAG?

**Retrieval-Augmented Generation (RAG)** adalah teknik yang meningkatkan jawaban AI dengan mengambil informasi relevan dari basis pengetahuan eksternal, seperti dokumen, database, atau API, sebelum menghasilkan respons.

Ini memungkinkan AI untuk melampaui data pelatihan statis yang digunakannya, mengurangi halusinasi dan memberikan jawaban yang terkini dan kaya konteks.

### Mengapa RAG Penting

LLM kuat, tetapi mereka hanya dapat merespons berdasarkan apa yang telah dilatih. Dengan RAG, Anda dapat menghubungkan mereka ke sumber data Anda sendiri: kebijakan perusahaan, makalah penelitian, atau dokumentasi pelanggan. Ini membuat mereka lebih berguna dalam skenario dunia nyata.

## Apa itu MCP?

**Model Context Protocol (MCP)** adalah **protokol komunikasi standar** yang mendefinisikan bagaimana aplikasi AI (klien) dapat berinteraksi dengan layanan eksternal (server).

Ini bukan model atau teknik, tetapi seperangkat aturan. MCP menggunakan JSON-RPC (2.0) untuk menangani permintaan dan respons, yang berarti setiap interaksi antara LLM (klien) dan server MCP mengikuti format JSON terstruktur untuk melakukan panggilan prosedur jarak jauh.

**Contoh permintaan dan respons JSON-RPC 2.0:**
```json
// Permintaan: Claude (klien MCP) meminta server MCP GitHub untuk membuat pull request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "create_pull_request",
  "params": {
    "repo": "fwwz/awesome-app",
    "branch": "feature-x",
    "title": "Tambah fitur baru"
  }
}

// Respons: Server MCP GitHub mengembalikan hasil
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "url": "https://github.com/fwwz/awesome-app/pull/42",
    "status": "success"
  }
}
```

Dengan kata lain:
- **RAG tentang konteks**: mengambil pengetahuan relevan.
- **MCP tentang konektivitas**: memberikan cara yang aman dan standar untuk mengakses sumber daya eksternal.

## Perbedaan Utama Sekilas

| Fitur | RAG (Retrieval-Augmented Generation) | MCP (Model Context Protocol) |
| :--- | :--- | :--- |
| **Konsep Inti** | Teknik untuk memberi dasar respons LLM dengan data eksternal. | Protokol untuk menghubungkan LLM ke alat, API, atau sumber data eksternal. |
| **Tujuan Utama** | Meningkatkan akurasi dan mengurangi halusinasi. | Memberikan jembatan yang aman, dapat diperluas, dan standar antara LLM dan sistem eksternal. |
| **Cara Kerja** | 1. Ambil dokumen relevan dari basis pengetahuan. <br /> 2. Tambahkan ke konteks model. <br /> 3. Hasilkan respons yang didasarkan. | 1. LLM (klien) mengirim permintaan JSON-RPC. <br /> 2. Server MCP memanggil layanan atau API yang mendasarinya. <br /> 3. Server mengembalikan hasil ke klien. |
| **Analogi** | Seorang siswa menulis laporan: pergi ke perpustakaan, baca referensi, lalu tulis. | Adaptor daya universal: sambungkan dengan aman ke stopkontak apa pun (alat atau API). |
| **Hubungan** | RAG adalah pola yang dapat menggunakan MCP sebagai fondasinya. | MCP adalah kerangka kerja yang dapat mengaktifkan RAG dan banyak aplikasi lainnya. |

## Cara Mereka Bekerja Bersama: Contoh

Bayangkan asisten AI yang menjawab pertanyaan tentang dokumentasi perusahaan Anda.

**Tanpa MCP**
Anda akan menulis kode kustom yang menghubungkan AI langsung ke database vektor. Ini rapuh, sulit dipelihara, dan menimbulkan risiko keamanan.

**Dengan MCP**
```txt
Pengguna: "Apa kebijakan liburan kami?"

Claude (klien MCP):
  → mengirim permintaan { "action": "search_docs", "params": {"query": "kebijakan liburan"} }

Server Dokumentasi MCP:
  → meminta basis pengetahuan internal
  → mengembalikan dokumen kebijakan HR yang relevan

Claude:
  → menggunakan konteks yang dikembalikan untuk menghasilkan jawaban yang didasarkan
```

Ini adalah sistem RAG yang dibangun di atas MCP.

## Contoh Nyata: GitHub dan Claude

MCP tidak terbatas pada RAG. Misalnya, jika Anda ingin Claude membuka pull request di GitHub:

```txt
Claude (klien MCP):
  → mengirim permintaan { "action": "create_pull_request", "params": {
      "repo": "fwwz/awesome-app",
      "branch": "feature-x",
      "title": "Tambah fitur baru"
    }}

Server GitHub MCP:
  → menerjemahkan permintaan menjadi panggilan API GitHub REST
  → otentikasi dengan aman
  → membuat pull request
  → mengembalikan tautan PR

Claude:
  → merespons dengan "Saya telah membuka pull request: <link>"
```

Alih-alih mengoding integrasi kustom untuk setiap LLM, MCP menyediakan antarmuka standar yang dapat digunakan oleh model kompatibel apa pun. Misalnya, begitu Anda memiliki server GitHub MCP, Claude, GPT-4, atau LLM lain yang diaktifkan MCP dapat berinteraksi dengan GitHub melalui titik akhir yang sama.

Demonstrasi praktis dari ini dapat ditemukan di [postingan Reddit ini](https://www.reddit.com/r/ClaudeAI/comments/1ixuiih/showed_off_claude_37_sonnet_in_a_vibe_sesh_to_my/?utm_source=share&utm_medium=web3x&utm_name=web3xcss&utm_term=1&utm_content=share_button), di mana Claude berinteraksi dengan API GitHub dengan lancar melalui MCP.

Sebagai alternatif, Anda dapat menggunakan model lokal dengan Ollama. [Artikel ini](https://medium.com/data-science-in-your-pocket/model-context-protocol-mcp-using-ollama-e719b2d9fd7a) memberikan panduan langkah demi langkah terperinci tentang cara menghubungkan Ollama dengan server MCP.

Standarisasi ini menunjukkan dua manfaat utama MCP:
1. **Interoperabilitas**: Tulis sekali, gunakan dengan LLM kompatibel apa pun
2. **Keamanan**: Semua interaksi API ditangani oleh server MCP, tidak secara langsung oleh LLM

## Kasus Penggunaan Praktis MCP

Server MCP dapat mengekspos banyak alat yang berbeda:

* **Alat Pemesanan Penerbangan**: berinteraksi dengan beberapa maskapai melalui satu antarmuka
* **Alat Kalkulator**: memastikan matematika dieksekusi dengan benar
* **Alat Cuaca**: mengambil pembaruan cuaca langsung dengan aman
* **Alat Kalender**: membaca dan memperbarui jadwal
* **Alat Sistem File**: menjelajahi dan mengedit file dengan aman di sandbox

Dengan mengabstraksi perbedaan API, MCP memungkinkan agen AI beroperasi lebih seperti skrip otomatisasi canggih yang dapat berpikir.

## Poin-Poin Utama

* **RAG** adalah teknik untuk memberi dasar respons LLM dengan data eksternal.
* **MCP** adalah protokol yang menstandarkan cara LLM terhubung ke alat dan data.
* Bersama-sama, mereka mengaktifkan agen AI yang akurat, dapat diperluas, dan aman.
* Meskipun API REST sudah ada, MCP memperkenalkan struktur konsisten yang dapat dipahami secara asli oleh LLM.

## Kesimpulan

RAG dan MCP bukan ide yang bersaing, mereka saling melengkapi.

* RAG berfokus pada *apa* yang dilakukan model: meningkatkan jawaban dengan konteks yang benar.
* MCP berfokus pada *bagaimana* model terhubung: dengan aman dan dengan cara yang standar.

Seiring agen AI berkembang, harapkan MCP memainkan peran yang semakin besar dalam menghubungkan LLM ke dunia nyata, dengan RAG sebagai salah satu aplikasinya yang paling penting.
